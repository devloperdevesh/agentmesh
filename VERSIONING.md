# Versioning Policy

AgentMesh follows Semantic Versioning.

Version format:

MAJOR.MINOR.PATCH

Example:

1.4.2

---

## MAJOR

Breaking API changes.

Example:

v1.x → v2.x

---

## MINOR

New features without breaking compatibility.

Example:

1.1 → 1.2

---

## PATCH

Bug fixes only.

Example:

1.2.1 → 1.2.2

---

## Release Process

1. Update CHANGELOG.md
2. Create release tag
3. Publish GitHub Release
4. Announce release
