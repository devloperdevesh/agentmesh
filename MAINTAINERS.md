# Maintainers

This document lists the maintainers responsible for guiding the development, technical direction, and community health of FaultPlane.

Maintainers help ensure that the project remains reliable, secure, and welcoming for contributors.

# Maintainer Responsibilities

Maintainers are responsible for:

## Code & Engineering

- Reviewing pull requests
- Maintaining code quality standards
- Reviewing architectural changes
- Ensuring technical consistency
- Guiding implementation decisions

---

## Project Governance

- Managing repository standards
- Maintaining documentation
- Reviewing RFCs and design proposals
- Managing project roadmap
- Coordinating releases

---

## Community

- Supporting contributors
- Helping resolve technical discussions
- Maintaining a respectful collaboration environment
- Improving contributor experience

---

## Security

Maintainers are responsible for:

- Reviewing security reports
- Coordinating vulnerability fixes
- Maintaining security documentation
- Following responsible disclosure practices

---

# Becoming a Maintainer

Maintainer roles are earned through consistent contribution and demonstrated technical leadership.

Contributors may become maintainers after demonstrating:

- High-quality technical contributions
- Strong understanding of the codebase
- Constructive code reviews
- Ability to make sound engineering decisions
- Commitment to project goals
- Long-term involvement with the community

---

# Maintainer Expectations

Maintainers should:

- Lead by example
- Make decisions based on technical reasoning
- Encourage open collaboration
- Keep discussions focused and constructive
- Prioritize project reliability and long-term sustainability

---

# Decision Making

Technical decisions should prioritize:

1. System reliability
2. Maintainability
3. Developer experience
4. Security
5. Long-term project direction

For major architectural changes, maintainers may request design discussions or RFC proposals before implementation.

---

# Emeritus Maintainers

Maintainers who are no longer actively involved may transition to emeritus status while retaining recognition for their contributions.

---

# Contact

For project governance questions, contributors may open a GitHub discussion or contact the maintainers through the project's official communication channels.
